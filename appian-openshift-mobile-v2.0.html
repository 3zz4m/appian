SECTION 01Core Concepts — Appian on OpenShift

Before touching a single command, make sure you have these concepts solid. Every production problem traces back to one of them.

1.1What is Appian & Why OpenShift?

Appian is an enterprise low-code platform for process automation and AI-powered workflows. Since version 25.4, Appian runs exclusively as a containerised workload on Kubernetes. OpenShift is the enterprise-grade, security-hardened Kubernetes distribution used in production.

All Appian microservices run as containerized pods managed by the Appian Operator — a Kubernetes-native controller with automated lifecycle management.

SECTION 02Prerequisites

Missing a prerequisite will silently block you hours into deployment. Verify every item before running a single command.

2.1Infrastructure Checklist

  • ☐Red Hat OpenShift 4.14+ cluster operational (bare-metal IPI/UPI or CRC for dev)
  • ☐Minimum 3 worker nodes (HA) or 1 node (SA/CRC) with 16+ vCPU and 64+ GB RAM each
  • ☐MetalLB or equivalent load balancer configured (bare-metal)
  • ☐cert-manager operator installed and running
  • ☐ODF (OpenShift Data Foundation) installed for CephFS/RBD storage (HA only)
  • ☐DNS configured for Appian URL and OpenShift routes
  • ☐TLS certificates provisioned (enterprise CA or self-signed for dev)

2.2Accounts & Downloads

  • ☐Red Hat account at console.redhat.com
  • ☐MyAppian account with active subscription
  • ☐AoK licence files requested (temporary 6-day + long-term)
  • ☐Appian Operator Helm chart downloaded (.tgz)
  • ☐Appian container images available (operator, webapp, data-server, search-server, service-manager, kafka, zookeeper, httpd)
  • ☐appian.yaml CR template downloaded from Appian docs

2.3External Database

  • ☐SQL Server 2022 (or MariaDB/MySQL/PostgreSQL) installed and running
  • ☐Appian database created with collation SQL_Latin1_General_CP1_CI_AS
  • ☐READ_COMMITTED_SNAPSHOT ON set on database
  • ☐DB user with db_owner for initial schema creation
  • ☐Database is network-reachable from pod CIDR on port 1433
  • ☐JDBC connection URL confirmed working
CRITICALLicence files are site-specific. k3.lic and k4.lic must match the site name and URL in your CR. A mismatch prevents the site from starting.

2.4Tools

  • ☐oc CLI installed and logged in to cluster
  • ☐Helm 3 installed
  • ☐Podman or Docker available for image management
  • ☐skopeo installed (air-gapped environments)

2.6Preflight Validation Script

Run this script before starting any deployment. All checks must pass:

bash · appian-preflight.sh
#!/bin/bash
# Appian on OpenShift — Pre-Deployment Health Check
echo "=== APPIAN OPENSHIFT PREFLIGHT ==="
# 1. Cluster connectivity
oc whoami >/dev/null 2>&1 || { echo "❌ Not logged in"; exit 1; }
echo "✅ Cluster: $(oc whoami --show-server)"
# 2. Storage classes
oc get storageclass --no-headers | grep -q . || { echo "❌ No StorageClass"; exit 1; }
echo "✅ StorageClasses: $(oc get sc --no-headers | wc -l) available"
# 3. cert-manager
oc get pods -n cert-manager --no-headers 2>/dev/null | grep -q Running || { echo "❌ cert-manager not running"; exit 1; }
echo "✅ cert-manager: Running"
# 4. Database reachable
SQL_HOST=${SQL_HOST:-"sql-server.example.com"}
SQL_PORT=${SQL_PORT:-1433}
nc -zv $SQL_HOST $SQL_PORT 2>&1 | grep -q succeeded || { echo "❌ DB unreachable"; exit 1; }
echo "✅ Database: $SQL_HOST:$SQL_PORT reachable"
# 5. DNS resolution
APPIAN_URL=${APPIAN_URL:-"appian.yourcompany.com"}
dig +short $APPIAN_URL | grep -q . || { echo "❌ DNS failure for $APPIAN_URL"; exit 1; }
echo "✅ DNS: $APPIAN_URL resolves"
# 6. Helm available
helm version --short >/dev/null 2>&1 || { echo "❌ Helm not installed"; exit 1; }
echo "✅ Helm: $(helm version --short)"
echo "✅ ALL PREFLIGHT CHECKS PASSED"
SECTION 03Air-Gap / Registry Mirror
AIR-GAP ONLYSkip this section if your cluster has internet access.

Air-gapped environments require all images to be pre-loaded into an internal registry.

3.1Mirror Images

bash · mirror-images.sh
#!/bin/bash
REGISTRY="registry.internal.corp:5000"
APPIAN_TAG="26.2.0"
SOURCE="registry.appian.com/appian"

for img in operator webapp data-server search-server service-manager kafka zookeeper httpd; do
  skopeo copy \
    docker://${SOURCE}/${img}:${APPIAN_TAG} \
    docker://${REGISTRY}/appian/${img}:${APPIAN_TAG} \
    --authfile /root/pull-secret.json
done
echo "✅ All images mirrored"

3.3Image Digest Verification

bash · verify-images.sh
#!/bin/bash
REGISTRY="registry.yourcompany.com:5000"
TAG="26.2.0.0"
for img in operator webapp data-server search-server service-manager kafka zookeeper httpd; do
  DIGEST=$(podman inspect ${REGISTRY}/appian/${img}:${TAG} --format '{{.Digest}}' 2>/dev/null)
  if [ -z "$DIGEST" ]; then
    echo "❌ MISSING: appian/${img}:${TAG}"
  else
    echo "✅ appian/${img}:${TAG} → ${DIGEST:0:20}…"
  fi
done
CRITICALCompare each digest against values published in Appian release notes. A mismatched digest means a corrupted or tampered image.

SECTION 04Operator Installation

The Appian Operator manages the full lifecycle. Install once per cluster.

4.1Namespace & Certificates

bash
oc create namespace appian-operator

# Create self-signed issuer for webhook TLS
cat <<EOF | oc apply -f -
apiVersion: cert-manager.io/v1
kind: Issuer
metadata:
  name: appian-operator-selfsigned-issuer
  namespace: appian-operator
spec:
  selfSigned: {}
EOF

# Create webhook certificate
cat <<EOF | oc apply -f -
apiVersion: cert-manager.io/v1
kind: Certificate
metadata:
  name: appian-operator-webhooks-certificate
  namespace: appian-operator
spec:
  secretName: appian-operator-webhooks
  issuerRef:
    name: appian-operator-selfsigned-issuer
  dnsNames:
  - appian-operator-webhooks.appian-operator.svc
  - appian-operator-webhooks.appian-operator.svc.cluster.local
EOF

oc get certificate -n appian-operator
# READY should show True

oc create secret docker-registry appian-pull-secret \
  -n appian-operator \
  --docker-server=registry.yourcompany.com:5000 \
  --docker-username=admin \
  --docker-password="$(cat /path/to/registry-password.txt)"

4.2Install via Helm

bash
helm install appian-operator \
  ./appian-operator-<version>.tgz \

6.4NetworkPolicy — Zero-Trust

yaml · networkpolicies.yaml
# 0. Allow DNS egress (apply FIRST)
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: appian-allow-dns-egress
  namespace: appian-prod
spec:
  podSelector: {}
  policyTypes: [Egress]
  egress:
  - to:
    - namespaceSelector:
        matchLabels:
          kubernetes.io/metadata.name: openshift-dns
    ports:
    - {protocol: UDP, port: 53}
    - {protocol: TCP, port: 53}
---
# 1. Default deny all
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: appian-default-deny
  namespace: appian-prod
spec:
  podSelector: {}
  policyTypes: [Ingress, Egress]
---
# 2. Allow ingress from router to webapp
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: appian-allow-ingress-webapp
  namespace: appian-prod
spec:
  podSelector:
    matchLabels: {app.kubernetes.io/component: webapp}
  policyTypes: [Ingress]
  ingress:
  - from:
    - namespaceSelector:
        matchLabels: {name: openshift-ingress}
    ports: [{protocol: TCP, port: 8080}]
---
# 3. Internal pod-to-pod + SQL egress
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: appian-allow-internal
  namespace: appian-prod
spec:
  podSelector: {}
  policyTypes: [Ingress, Egress]
  ingress:
  - from: [{podSelector: {}}]
  egress:
  - to: [{podSelector: {}}]
  - ports: [{protocol: TCP, port: 1433}]
bash

SECTION 12Upgrades

Always upgrade Operator first, then the site. Never both at the same time.

12.0Pre-Upgrade Gate Script

bash · pre-upgrade-gate.sh
#!/bin/bash
NS=${1:-appian-prod}
echo "=== PRE-UPGRADE GATE ==="
SM_POD=$(oc get pod -n $NS -l app.kubernetes.io/component=service-manager -o name | head -1)
oc exec -n $NS $SM_POD -- /usr/local/appian/ae/bin/checkpoint.sh || { echo "❌ Checkpoint FAILED"; exit 1; }
echo "✅ Engine checkpoint"
velero backup create pre-upgrade-$(date +%Y%m%d) --include-namespaces=$NS --snapshot-volumes --wait || { echo "❌ Backup FAILED"; exit 1; }
echo "✅ Velero backup"
UNHEALTHY=$(oc get pods -n $NS --no-headers | grep -v Running | grep -v Completed | wc -l)
[ "$UNHEALTHY" -eq 0 ] || { echo "❌ $UNHEALTHY pods not Running"; exit 1; }
echo "✅ All pods Running"
echo "✅ ALL GATES PASSED — Safe to upgrade"

12.1Upgrade Procedure

1. Pre-upgrade checkpoint
Run checkpoint.sh, DB backup, Velero backup
2. Pause MachineHealthCheck
oc annotate mhc ... cluster.x-k8s.io/paused=""
3. Upgrade Operator
helm upgrade appian-operator ./new-chart.tgz
4. Update CR version
oc patch appian ... -p '{"spec":{"version":"NEW"}}'
5. Monitor & validate
Watch rolling update, verify health, run sp_updatestats
6. Resume operations
Resume MHC, revoke temp DB permissions
